        Trap/Reclaim Patterns Glossary
        Bull Trap (Failed Breakout): VIX breaks above resistance but quickly reverses back below, trapping traders who bought the breakout.
        Bear Trap (Failed Breakdown): VIX breaks below support but quickly reverses back above, trapping traders who sold the breakdown.
        Support Reclaim: VIX recovers and closes back above a support level after previously breaking below it, suggesting the support is holding.
        Resistance Reclaim: VIX falls back and closes below a resistance level after previously breaking above it, suggesting the resistance is holding.

        Not Financial Advice

        This dashboard is for research only. Signals can be wrong, delayed, or incomplete. You are responsible for risk management and execution decisions.

        Quick Start (2 Minutes)

        1. Start here. Read the risk note above so the dashboard is used as research support, not as a trading instruction.
        2. Open Daily Report after the nightly run completes.
        3. Review Tonight's 5 Key Changes first for the fast market summary.
        4. Check YOLO Changes for new patterns and patterns no longer detected.
        5. Use Confluence Score and Sector Heatmap to prioritize candidates.
        6. Click any ticker to jump into Chart + Levels and inspect structure.
        7. Use Bollinger Bands toggle when you want volatility context.
        8. Add names to Watchlist so you can revisit the same basket quickly.

        What Each Tab Does

        Guide
        How to use the app, read signals, and avoid common mistakes.
        Daily Report
        Nightly summary of what changed, strongest setups, VIX analysis context, and YOLO deltas.
        Chart + Levels
        Per-ticker chart with support/resistance, gaps, trendlines, rule/hybrid overlays, candlestick signals, optional Bollinger Bands, and YOLO boxes.

        Daily Report Read Order

        1. Tonight's 5 Key Changes: one-screen summary of what materially changed.
        2. Confluence Score: start in compact view, then open More Metrics only when you need to audit volatility and aging inputs. The top line is the current leader or leader cluster, not the only tradable name.
        3. Sector Heatmap: identify leadership/laggards and rotation.
        4. YOLO Changes vs Previous Run: actionable delta, not raw count noise.
        5. Large Moves + 52-week Extremes: momentum continuation or exhaustion candidates.

        How Confluence Score Is Built

        1. Step 1: classify the setup family. The report first asks what the chart is trying to do now: bullish reversal, bullish continuation, bearish reversal, bearish continuation, or neutral watch.
        2. Step 2: weight fresh information first. The latest active YOLO closest to the most recent close matters most. Recent persistent YOLO matters next. Older YOLO is downgraded to context, not treated as a fresh trigger.
        3. Step 3: look for confirmation. Candlestick confirmation, trend state, MA reclaims/crosses, support/resistance location, gaps, breakouts/breakdowns, stretch, and volume all raise or reduce conviction.
        4. Step 4: use valuation as context, not as direction. Discount and PEG can improve or weaken the case, but cheap alone does not make a bullish setup and expensive alone does not make a bearish one.
        5. Step 5: assign the tier. A/B requires aligned evidence. C/D means weaker alignment, stale context, or missing confirmation. Several names can be valid at the same time; use the chart to decide which one has the cleanest current tape.

        Chart + Levels Workflow

        1. Load ticker from report or search box.
        2. Mark location versus primary/secondary support/resistance.
        3. Check open gaps and trendlines for context.
        4. Use YOLO Audit to see whether the latest pattern is fresh, recent and persisting, or just stale context.
        5. Enable Rule/Hybrid Lines and YOLO Patterns to compare machine + rule signals.
        6. Turn on Bollinger Bands if you want to gauge stretch/squeeze conditions.
        7. Decide only when multiple signals agree (structure + momentum + valuation).

        Chart Toggles (How to Use)

        • Trendlines: reveal support/resistance line geometry from recent pivots.
        • YOLO Boxes: draw the latest active YOLO box on the chart. This is current-context only, not full history.
        • Rule/Hybrid Lines: show rule-based structure when available. If the toggle says (none), there is no usable overlay for that ticker/timeframe.
        • Bollinger Bands: optional volatility envelope (20, 2) around price.
        • YOLO Patterns: AI-detected pattern labels. Always read the age and audit table before treating one as actionable.
        • Bars D/W: switch between daily and weekly candles for timeframe alignment.

        Signal Glossary (Short)

        YOLO Patterns
        • Computer-vision detections on candlestick images (daily/weekly timeframe).
        • x0_date/x1_date define the detected pattern window.
        • age_days shows how old the pattern end-date is relative to run date.
        • Fresh YOLO can contribute to actionability. Recent persistent YOLO is useful secondary evidence. Stale YOLO is context only.
        • Use YOLO Changes to identify truly new or invalidated patterns.
        Valuation Fields
        • Discount %: upside/downside versus analyst target.
        • PEG: P/E adjusted for growth; lower can indicate better value for growth.
        • Deep Value preset is stricter than Undervalued.
        Technical Structure
        • Support: area where buyers previously stepped in.
        • Resistance: area where sellers previously stepped in.
        • Gaps: unfilled zones often retested later.

        Run Schedule

        • Daily pipeline: weekdays at 22:00 UTC (ingest → YOLO → report).
        • Weekly YOLO pass: Saturday (regenerates report after run).
        • Report tab always shows the latest completed run; no manual refresh button is required.

        Common Mistakes

        • Trading one signal in isolation.
        • Ignoring timeframe mismatch (weekly pattern vs short-term trade horizon).
        • Treating old patterns as fresh without checking age_days.
        • Ignoring liquidity/volume when evaluating breakouts.
